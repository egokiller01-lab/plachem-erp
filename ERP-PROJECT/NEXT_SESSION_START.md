# [Next Session Start] Phase 4: 생산 고도화 및 BOM 연동

본 문서는 보안 및 UI 안정화 작업을 마친 후, 다음 단계인 **생산 모듈 고도화**를 즉시 시작하기 위한 가이드입니다.

## 1. 현재 시스템 상태 (Handoff Recap)
- **보안**: 16개 핵심 테이블 RLS 적용 완료 및 UI `canEdit` 제어권 확보.
- **재고/원가**: MAC 계산 엔진 완비 및 월마감/조정 기능 안정화.
- **생산 (현황)**: 기본 생산 실적 입력 기능은 있으나, 하위 자재 자동 차감 및 BOM 연동은 미비한 상태.

## 2. Phase 4 핵심 과제
### [A] BOM(Bill of Materials) 체계 구축
- `BOM_headers` (제품별 표준 배합비 헤더)
- `BOM_items` (투입 원부재료 및 표준 소요량)
- **목표**: 생산 실적 입력 시 BOM을 참조하여 원부재료 소요량을 자동 제안.

### [B] 생산 입출고 로직 자동화
- 생산 실적(Production Output) 발생 시 관련 자재(Production Input) 자동 트랜잭션 생성.
- **연계**: `inventory_transactions`와 실시간 MAC 엔진 연동.

### [C] 외주 가공비 및 MAC 반영
- 외주 생산 시 발생하는 가공비를 제품 원가(MAC)에 가산하는 로직 구현.

## 3. 다음 작업 시작점 (Start Point)
1. **DB 스키마 점검**: 기존 `production_inputs`, `production_outputs` 테이블과 BOM 테이블 간의 관계 설계.
2. **생산지시 UI 설계**: BOM을 로드하여 자재 배정 및 소요량을 자동으로 뿌려주는 입력 화면 고도화.
3. **Trigger/RPC 구현**: 생산 확정 시 자재 차감 및 제품 입고를 원자적으로 처리하는 서버 사이드 로직.
